# 文库 Pooling 表自动化处理操作文档

## 涉及文件

| 文件 | 说明 |
|------|------|
| `江西华大T7+上机-PE150-20260711.xlsx` | A表，数据源 |
| `20260711文库pooling表T7+PE150-zss.xlsx` | B表，输出目标 |
| `外包文库质检总表2026(1).xlsx` | C表，外包文库质检数据 |
| `2026年1月-12月自建库出库报告-T7.xlsx` | D表，自建库出库数据 |

## 执行顺序

```
clear_pooling.py → step1_migrate.py → step2_lookup.py → step3_group_sort.py 
→ step4_formula.py → step5_lane.py → step6_t7_prepare.py → format_font.py
```

---

## 步骤一：数据迁移（A表 → B表）

**脚本：** `step1_migrate.py`

**操作对象：** B表 A/B/C 工作表

**规则：**

| # | 规则 | 
|---|------|
| 1 | A表 C列(HaploX编号) → B表 B列(文库编号) |
| 2 | A表 C列为空 → A表 D列(样本名称) → B表 B列 |
| 3 | A表 B列或C列出现合并格池文库编号 → 池文库编号 → B表 B列 |
| 4 | A表 J列(预分配数据量G) → B表 D列(数据量G) |
| 5 | B表 B列为池文库编号 → A表池文库 J列求和 → B表 D列 |
| 6 | A表 H列(文库类型) → B表 G列 |
| 7 | A表 L列(客户单位) → B表 H列 |
| 8 | A表 M列(备注) → B表 L列 |

**去重：** B列相同文库编号保留首行，后续重复行丢弃。

## 步骤二：查找填充（C表/D表 → B表）

**脚本：** `step2_lookup.py`

**操作对象：** B表 A/B/C 工作表

**匹配逻辑：**
- B列以 `HGC-Lib-` 或 `HGC-POOL-` 开头 → C表 F列(HGC编号)匹配
- 其他 → D表 D列(样本编号)匹配

### HGC类型（C表匹配）

| C表列 | → | B表列 |
|--------|---|------|
| L列(Qubit浓度) | → | C列 + N列 |
| V列(文库结构) | → | O列 |
| W列(磷酸化*) | → | P列 |
| X列(环化*) | → | Q列 |
| O列(板号) | → | K列 |
| 平均片段(bp) | → | M列 |

### 非HGC类型（D表匹配）

| D表列 | → | B表列 |
|--------|---|------|
| K列(Qubit浓度) | → | C列 + N列 |
| O列(结果评价) | → | O列 |
| T列(版号) | → | K列 |
| S列(孔位) | → | J列 |
| 平均片段(bp) | → | M列 |

## 步骤三：分组与排序

**脚本：** `step3_group_sort.py`

**操作对象：** B表 A/B/C 工作表

### 1. 大类判断

| 条件 | 分类 | 位置 |
|------|------|------|
| G列 或 O列 含"华大" 或 P列="已磷酸化" | 直接环化文库 | 后面 |
| 其他 | 转化文库 | 前面 |

### 2. 分组规则

- 先按客户单位(H列)分组，再按文库类型(G列)细分
- 同类内按（客户单位 + 文库类型）排序聚合

**特殊合并：**
- 广州基迪奥科技服务有限公司 = 广州奥智生物科技有限公司（视为同一客户）
- G列为 "XXX-T7" 视为同一客户

**池文库处理：**
- HGC-POOL- 开头的池文库：同客户+同类型合并为一组（不与 HGC-Lib- 混合）

### 3. D_sum > 1000 拆分

组内 D列总和 > 1000 → 拆成 ceil(N) 个子组，行数尽量均匀。

## 步骤四：公式计算、重分组、汇总

**脚本：** `step4_formula.py`

**操作对象：** B表 A/B/C 工作表

### 公式定义

| | 转化文库 | 直接环化文库 |
|------|----------|-------------|
| 总摩尔质量 | 50 nM | 300 ng |
| E列(文库摩尔质量) | D/组D合计×50 | D/组D合计×300 |
| F列(最终取样体积) | =ROUND(E/C×G汇总,3) | 同左 |
| I列(TE Buffer) | =ROUND(20-F汇总,3) | =ROUND(40-F汇总,3) |

### 重分组

触发条件：D_sum > 1000 **或** max/min > 5
- 按F升序排列
- 以 max/min > 3 为界切分新组
- 子组内按B列升序
- 重新计算E/F

### 体积调整

组内 F < 0.250 → 找最小整数N使 `min×N > 0.500` → 倍数N写入汇总行G列

### 汇总行

| 列 | 内容 |
|----|------|
| D | =ROUND(SUM(D范围),2) |
| E | 组E合计（静态） |
| F | =IF(G>1,ROUND(SUM/G×2,3),ROUND(SUM,3)) |
| G | 缩放倍数N |
| I | TE Buffer（浅蓝背景） |
| M | =AVERAGE(M范围) |

## 步骤五：Lane编号、合并、框线、文库环化

**脚本：** `step5_lane.py`

### A/B/C 工作表

- A列 Lane编号：A1,A2... / B1,B2... / C1,C2...
- 每数据行 A列 合并单元格
- A列~K列 加框线（汇总行不加）
- 直接环化组 A列 标记红色

### 文库环化 工作表

| 列 | 来源/公式 |
|----|----------|
| A | Lane编号 |
| D | =C×0.66×1 |
| E | 数据量（组内D列求和） |
| F | =D |
| G | =F/B |
| H | 文库类型（组内第一行G列） |
| I | 客户单位（组内第一行H列） |
| J | =40-G |
| M | =22×L/F |
| B/C/L | 留空（用户填入） |

## 步骤六：T7+制备

**脚本：** `step6_t7_prepare.py`

### 数据传递

| 文库环化 | → | T7+制备 |
|---------|---|--------|
| A列(文库编号) | → | B列(文库ssDNA编号) |
| E列(数据量) | → | H列 |
| H列(文库类型) | → | N列 |
| I列(客户单位) | → | O列 |

### A列 laneID

格式：`MMDD + Lane首字母`，如 `0717A`。相同laneID为一组。

### 公式

| 列 | 公式 |
|----|------|
| E | D与组中位值比较（差距≤100→"正常"，超→"偏大/偏小Xbp"） |
| F | =D×0.33×G×0.001 |
| G | =900×L |
| I | =ROUND(H/H总,4) |
| K | =I×J |
| L | =K/K总 |
| M | =F/C |
| P(汇总) | =96-M汇总 |
| Q | =I×J |

### 汇总行（每组第一空白行）

| 列 | 公式 |
|----|------|
| D | =MEDIAN(D范围) |
| H | =SUM(H范围) |
| K | =SUM(K范围) |
| M | =SUM(M范围) |
